Witaj na moim blogu! Dzisiaj rozpoczniemy naszą przygodę z Django, dowiesz jak Django zainstalować, jak skonfigurować project oraz stworzymy pierwszą aplikację. Do dzieła!

Czego potrzebujemy?

Oczywiście edytora kodu źródłowego lub środowiska programistycznego (IDE), lista edytorów to na przykład Sublime Text, Atom, Notepad++. Ze środowisk programistycznych możemy wybrać Visual Studio Code albo PyCharm. Ja polecam ten drugi, ponieważ od początku do końca jest dedykowany Pythonowi, a wersja Professional (kosztuje 200 dolarów rocznie, ale jeśli jesteś studentem to możesz aplikować o darmową wersję Professional, osobiście sprawdziłem i działa) zapewnia natywne wsparcie dla narzędzi takich jak Django, Flask czy Twisted, w tym mamy np. podpowiadanie składni w szablonach.

Oprócz tego potrzebujemy samego Pythona (zakładam, że to już masz zainstalowane) oraz instalator pakietów do niego - Pip. To już mamy ogarnięte, teraz potrzebujemy zainstalować Django, robimy to komendą pip install django. Aby sprawdzić, czy wszystko zainstalowało się poprawnie, po wpisaniu komendy python -m django --version do terminala, powinieneś ujrzeć wersję Django, którą zainstalowałeś.

Tworzenie projektu

Przechodzimy do stworzenia i konfiguracji projektu. W celu jego utworzenia, wpisujemy komendę django-admin startproject [nazwa_projektu]. Nasz pierwszy projekt nazwę myfirstproject, więc odpowiednio pełna treść komendy będzie: django-admin startproject myfirstproject. Jak widzimy, utworzony został katalog o nazwie myfirstproject, a w jego wnętrzu plik manage.py oraz kolejny katalog o takiej samej nazwie. Tak naprawdę nazwa katalogu zewnętrznego nie ma dla Django znaczenia, możemy zmienić jego nazwę jeśli chcemy. Poniżej omówimy sobie czym są pliki w katalogach.

- manage.py - narzędzie wiersza poleceń służące do interakcji z projektem, to za jego pomocą będziemy uruchamiać loklany serwer czy wykonywać migracje bazy danych
- myfirstproject/__init__.py - pusty plik mówiący o tym, że ten katalog powinien być traktowany jako pakiet (czyli można go zaimportować)
- myfirstproject/settings.py - przechowuje ustawienia naszego projektu, w przyszłości omówimy sobie część z nich
- myfirstproject/urls.py - plik z deklaracjami adresów URL naszego projektu, to w nim będziemy przypisywać adresy URL do poszczególnych aplikacji (o aplikacjach poniżej)
- myfirstproject/asgi.py - punkt wejścia dla serwera ASGI naszego projektu
- myfirstproject/wsgi.py - punkt wejścia dla serwera WSGI naszego projektu

Teraz sprawdzimy czy wszystkie pliki są na swoim miejscu i działają, wpisz komendę python manage.py runserver, powinieneś ujrzeć taki tekst: Performing system checks... System check identified no issues (0 silenced). You have unapplied migrations; your app may not work properly until they are applied. Run 'python manage.py migrate' to apply them. January 14, 2021 - 15:50:53 Django version 3.1, using settings 'myfirstproject.settings' Starting development server at http://127.0.0.1:8000/ Quit the server with CONTROL-C.
Uruchomiłeś serwer dostarczony przez Django. Nasłuchuje on na adresie "127.0.0.1" i porcie "8000". Serwer ten jest o tyle fajny, że za każdym razem kiedy poczynisz zmiany w kodzie ten natychmiast się automatycznie restartuje.

Tworzenie aplikacji

Czym się różni projekt od aplikacji? Projekt to zbiór aplikacji i ich konfiguracji. Aplikacja to część projektu, która "coś" robi, np. aplikacja może być blogiem, może być systemem zarządzania treścią. Jeden projekt może mieć wiele aplikacji, jedna aplikacja może być w wielu projektach. Gdy jesteśmy w katalogu myfirstproject (tym zewnętrznym, tam gdzie mamy plik manage.py), do utworzenia aplikacji używamy komendy python manage.py startapp [nazwa_aplikacji]. Naszą aplikację nazwiemy "ankiety". Jak widzimy, utworzony został nowy katalog o nazwie "ankiety"

Pisanie pierwszego widoku
